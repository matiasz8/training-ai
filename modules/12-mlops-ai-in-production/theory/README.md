# Theory — MLOps & AI in Production

## 1) ¿Qué es MLOps?
Conjunto de prácticas para construir, desplegar y operar modelos de ML de forma confiable, reproducible y escalable.

## 2) Ciclo de vida típico
- Desarrollo y experimentación.
- Entrenamiento y validación.
- Registro de modelo.
- Despliegue.
- Monitoreo y retraining.

## 3) Componentes clave
- Versionado de datos, código y modelos.
- CI/CD para ML.
- Serving (batch, online, streaming).
- Observabilidad (latencia, drift, calidad de predicción).

## 4) Monitoreo
- Data drift.
- Concept drift.
- Degradación de métricas.
- Alertas y umbrales.

## 5) Estrategias de despliegue
- Canary.
- Blue/green.
- Shadow deployment.
- Rollback seguro.

## 6) Buenas prácticas
- Empezar simple y automatizar gradualmente.
- Definir SLOs desde el inicio.
- Mantener trazabilidad completa.
- Diseñar runbooks para incidentes.

## 7) Checklist de comprensión
- ¿Podés describir el flujo desde experimento a producción?
- ¿Sabés qué monitorear además de accuracy?
- ¿Podés definir una estrategia de rollback?
